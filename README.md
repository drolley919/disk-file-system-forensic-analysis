# Disk & File System Forensic Analysis (NTFS)

## Overview
This project documents a disk and NTFS file system forensic investigation conducted using FTK Imager as part of DFCS 605. The objective was to analyze file system artifacts, metadata, timestamps, and user-generated files to identify evidence relevant to a forensic inquiry.

The investigation demonstrates proper forensic methodology, evidence handling, and analytical reporting consistent with industry best practices.

## Scope of Analysis
- NTFS file system structure
- File metadata and timestamps
- User file artifacts
- Evidence integrity and validation
- Forensic documentation and reporting

## Tools Used
- FTK Imager
- Windows NTFS file system
- Cryptographic hashing (MD5/SHA)
- Native system artifact review

## Repository Structure
- `Executive-Summary.md` – High-level findings and conclusions
- `Evidence.md` – Evidence sources and collection methodology
- `Analysis.md` – Detailed forensic examination
- `Findings.md` – Analytical conclusions
- `Recommendations.md` – Security and forensic recommendations
